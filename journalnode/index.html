Semantic Alpha

Each frontier LLM encodes a different compression of human financial knowledge. Querying multiple models with identical market data produces novel alpha uncorrelated with traditional quantitative signals. When independent models converge on a directional thesis, that convergence is itself a measurable signal.

This is the LLM Consensus Hypothesis: reliability-weighted convergence of model outputs — measured by the Semantic Conviction Score (SCS) — correlates positively with forward price movement, particularly when inter-model variance is low. Journal Node operationalizes this thesis into a continuous analytical pipeline with on-chain provenance.

Four Analytical Modes

Mode A — Three-Agent Debate

A Bull Catalyst Scout and Bear Risk Auditor produce directional scores, then a Semantic Arbiter synthesizes both into a balanced SCS. If the bull/bear spread exceeds 60, recursive research triggers a verification pass. Output: directional conviction score with structured bull/bear rationale.

Mode B — Peer-Anchored Valuation

Identifies four peer companies, then queries up to 8 models to estimate fair market value. Outlier detection flags estimates beyond 1 standard deviation. Output: Valuation Anchor Composite (VAC) — a reliability-weighted trimmed mean of model estimates.

Mode C — Stochastic Stress Test

A single model is queried 5 times at varying temperatures (0.2 – 0.7) to measure output consistency. The Stability Coefficient (CV) classifies results as HIGH_STABILITY, LOW_RELIABILITY, or UNRELIABLE. Output: conviction stability score and temperature-sensitivity profile.

Mode D — Vision + Contrarian Trap

A vision-capable model transcribes chart screenshots into structured JSON, then up to 8 models provide technical recommendations. A trap detector evaluates price/volume divergence, RSI exhaustion, upper wicks, and stop-run patterns. Output: directional call with trap probability score.

Trade Ticket Workflow

Opening a trade creates an immutable ticket capturing the asset, direction, entry price, mode used, model consensus, and SCS score. Five fields lock permanently at creation — modelConsensus, scsScoreAtEntry, modeUsed, userAlignment, and signalPriceAtEntry — ensuring the accuracy feedback loop operates on honest, unaltered data.

Every analytical output receives a deterministic SHA-256 hash anchored to the Post Fiat network as a memo transaction — tamper-proof provenance proving when the analysis occurred and that its content is unaltered. On closure, each ticket's outcome updates the contributing models' reliability scores, creating a flywheel that compounds signal quality over time.

PFT Token Utility

PFT implements a performance-based fee model aligned with user outcomes. The system profits only when users profit cumulatively.

  • Performance Fee — 5% of profitable signal moves, charged only when a signal proves correct within its observation window.
  • High-Water Mark — Users with a net-negative cumulative track record pay zero fees. Charges resume only after returning to net-positive.
  • Timeframe Windows — Signals remain open for evaluation based on trade horizon: 24h for 1D, 7 days for 1W, 30 days for 1M.
  • On-Chain Anchoring — PFT is consumed to anchor trade tickets and SHA-256 hashed outputs as memo transactions on the Post Fiat network.
